# PRD: Правовой советник — Законодательство РК

**Тип документа:** Product Requirements Document (PRD)  
**Версия:** 1.0  
**Дата:** 24 июня 2026 г.  
**Статус:** Готов к разработке

---

## 1. Обзор продукта

### 1.1 Название
**Правовой советник** — ИИ-инструмент для правового анализа ситуаций по законодательству Республики Казахстан.

### 1.2 Назначение
Веб-приложение, позволяющее пользователю описать правовую ситуацию, задать конкретный вопрос и получить структурированный юридический анализ с указанием применимых норм законодательства Казахстана.

### 1.3 Целевая аудитория
- Физические лица, столкнувшиеся с правовыми проблемами
- Малый и средний бизнес, не имеющий штатного юриста
- Студенты юридических специальностей
- Специалисты смежных областей (HR, бухгалтеры, предприниматели)

---

## 2. Проблема и решение

### 2.1 Проблема
Получение первичной правовой консультации в Казахстане требует значительных временных и финансовых затрат. Большинство граждан не знают, к каким конкретным нормам законодательства относится их ситуация, и не могут самостоятельно найти нужную статью среди тысяч нормативных актов.

### 2.2 Решение
Приложение использует языковую модель (Claude Sonnet 4.6) с системным промптом, настроенным на роль казахстанского юриста, для мгновенного первичного анализа и указания конкретных норм права — без необходимости записи на приём или оплаты консультации.

---

## 3. Функциональные требования

### 3.1 Ввод данных

| Поле | Тип | Обязательно | Ограничения |
|---|---|---|---|
| Отрасль права | Выпадающий список | Нет | 10 предустановленных категорий |
| Описание ситуации | Многострочный текст | Да | До 2000 символов |
| Правовой вопрос | Многострочный текст | Да | До 500 символов |

**Поддерживаемые отрасли права:**
1. Гражданское право
2. Трудовое право
3. Семейное право
4. Уголовное право
5. Административное право
6. Налоговое право
7. Жилищное право
8. Земельное право
9. Корпоративное право
10. Защита прав потребителей

### 3.2 Обработка запроса

- Запрос отправляется в Anthropic API (`claude-sonnet-4-6`, `max_tokens: 1000`)
- Системный промпт задаёт роль казахстанского юриста и структуру ответа
- Ответ обрабатывается и форматируется на стороне клиента

### 3.3 Структура ответа

Ответ должен содержать следующие разделы:

1. **Правовая квалификация ситуации** — определение типа правоотношений
2. **Применимые нормы законодательства РК** — конкретные статьи, пункты, части нормативных актов
3. **Права и обязанности сторон** — что вправе и обязан делать каждый участник
4. **Рекомендуемые действия** — практические шаги для пользователя
5. **Ответственность за нарушение** — санкции (если применимо)
6. **Вывод** — краткое итоговое заключение

### 3.4 Выделение норм права

Ссылки на законодательство (статьи, пункты, кодексы, законы) автоматически выделяются визуально в тексте ответа с помощью тегов-меток (`.norm-tag`).

## 4. Нефункциональные требования

### 4.1 Производительность
- Время ответа API: ≤ 10 секунд в нормальных условиях
- Отображение состояния загрузки во время ожидания ответа

### 4.2 Интерфейс
- Двухколоночный макет: ввод слева, ответ справа
- Адаптивный дизайн: одноколоночный вид на мобильных устройствах (≤ 800px)
- Визуальный счётчик символов для полей ввода
- Поддержка горячих клавиш: Ctrl+Enter для отправки запроса

### 4.3 Доступность
- Чёткие подписи полей и подсказки
- Видимое состояние фокуса
- Отображение ошибок в явной форме (не скрытые)

### 4.4 Безопасность
- API-ключ Anthropic не передаётся в клиентский код (управляется на уровне платформы)
- Пользовательские данные не сохраняются между сессиями

## 5. Технические характеристики

### 5.1 Стек
- **Тип приложения:** Single-page HTML (без фреймворков, без сборщиков)
- **Шрифты:** Playfair Display (заголовки), Inter (интерфейс) — Google Fonts
- **API:** Anthropic `/v1/messages` — вызов непосредственно из браузера
- **Хранение данных:** отсутствует (только состояние в DOM)

### 5.2 Системный промпт

```
Ты — квалифицированный юрист, специализирующийся на законодательстве 
Республики Казахстан. Твоя задача — дать чёткий, структурированный 
правовой анализ ситуации.

Правила:
1. Отвечать строго по законодательству Казахстана
2. Указывать конкретные нормы: статья, часть, пункт, полное название акта
3. Структурировать ответ по 5 разделам (квалификация → нормы → права → 
   действия → ответственность)
4. Делать краткий вывод в конце
5. Указывать на необходимость срочного обращения в органы (если применимо)
6. Формулировать позицию чётко, без уклончивых ответов
7. Отвечать на русском языке
```

### 5.3 Параметры API

| Параметр | Значение |
|---|---|
| `model` | `claude-sonnet-4-6` |
| `max_tokens` | `1000` |
| Язык ответа | Русский |
| Формат ответа | Markdown (заголовки `##`, жирный `**`) |

Текст отказа от ответственности, отображаемый в интерфейсе:

> **Важно:** Данный инструмент использует ИИ для информационных целей и не заменяет консультацию квалифицированного юриста. Ссылки на нормы законодательства носят ориентировочный характер — всегда проверяйте актуальную редакцию на портале **adilet.zan.kz**.

---

## 8. Ограничения текущей версии

- Нет сохранения истории запросов
- Нет экспорта ответа (PDF, DOCX)
- Нет поддержки казахского языка (только русский)
- Нет ссылок на конкретные редакции нормативных актов (только наименования)
- Ограничение ответа: 1000 токенов (сложные ситуации могут быть обрезаны)

## 10. Критерии приёмки

- [ ] Пользователь может описать ситуацию и получить ответ менее чем за 15 секунд
- [ ] Ответ содержит минимум одну конкретную ссылку на норму законодательства РК
- [ ] Нормы права визуально выделены в тексте ответа
- [ ] Интерфейс корректно отображается на мобильных устройствах
- [ ] При ошибке API отображается понятное сообщение (не технический стек-трейс)
- [ ] Кнопка заблокирована во время ожидания ответа (предотвращение дублирования)
